Working with Buttons and Forms

Lab 6

What are Buttons and Forms?

Buttons are seen on all well-known webpages, as they are a necessity for a diverse and creative page. Buttons can be used for navigation, redirection, submission, resetting, and many other jobs. Buttons pair hand-in-hand with forms.
Forms are used to collect user data and send it to the server to process. Whenever you have typed in your login information, sent an application, or have clicked submit after typing anything, that is a form.

Custom Elements

Lab 5

What are Custom Elements?

Unlike HTML, custom elements allows you to change the appearance and behaviour of your code to fit your personal preference. Many web developers use custom elements for common features such as customized buttons, clocks, copyright dates, product display, and notifications, but the options span far further than that.

StyleSheets

Lab 4

What are StyleSheets?

Style sheets are used to display the visual aesthetic of your HTML. Some common uses are colour, font-size or weight, font-family, padding, and display-block. Style sheets can be used in three different ways:

  • Inline

    • Inline stylesheets are styles typed directly into the tag. An example of this could look like: 

    <p style="color: aqua;">text here</p>

  • Internal

    • CSS can also be within your code, inside it's own tag, known as <style>. Within the <style> tag, you can create as many visual changes as you wish, with the layout being a bit different. When your CSS code is within the <style> tag, you need to specify what HTML tag is receiving the change. It would look like this: 

    <style>
    h1 {
        color: green;
}
</style>
  • External

    • The final way CSS can be used is externally. Most professional web developers will create a separate style sheet–a .css file–and link it to their index page. This helps prevent clutter on the page they are writing most of their code. It often looks like this: 

    <link rel="stylesheet" href="styles.css">
